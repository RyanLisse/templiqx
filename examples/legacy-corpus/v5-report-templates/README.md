# Legacy v5 report templates (real, format reference)

Three real legacy Basenet **v5 report templates** — the concrete source
artifacts the report engine (BLI-230) must interoperate with. Unlike the
synthetic fixtures under [`../fixtures/`](../fixtures/), these are **not**
generated by `tools/templiqx-legacy-docx-fixtures` and are **not** scanned by
the conformance suite. They are reference evidence only: they document *which
wire formats and authoring dialect* still exist in production, not a support
claim.

They are safe to track: each was inspected and contains **only merge-field /
Velocity template logic — no client data** (`#foreach`, `#if`, `$func.util…`,
`$comp.name`, `$dossier.pshdescr`). The one trace of PII is document authorship
metadata (last-saver name) in the `.xls`, which is normal template metadata.

| File | v5 "power" | Format | Dialect evidence |
|------|-----------|--------|------------------|
| `Bedrijfsoverzicht.rtf` | Company overview | RTF | `#foreach($comp in $data.companys)`, `$comp.name`, `$comp.tel1`, `#if(!$func.util.isEmptyOrZero(...))` |
| `Dossiervoorblad.docx` | Case cover sheet | DOCX (Velocity-in-MERGEFIELD) | `#foreach($dossier in $data.dossiers)`, `$dossier.pshdescr`, `$client.company`, `#if($client.rtype != 'Particulier')` |
| `OHW-per-kostenplaats.xls` | WIP per cost centre | BIFF `.xls` | `#foreach($kostenplaats in $factuurregelsOhwPerKostenplaats)`, `$func.util.sum($project.entries,'priceExcl')` |

## What these evidence (and what they do not)

These prove the **formats and authoring style** are real. They do **not** imply
Templiqx executes Velocity/`$func`, parses BIFF `.xls`, or reproduces v5 layout
fidelity. See [`docs/guides/report-engine-compatibility.md`](../../../docs/guides/report-engine-compatibility.md)
for the format-evidence table and the v5-four-powers → Templiqx support matrix.

Templiqx's stance:
- **DOCX** — bounded `templiqx-docx-v5` migration + synthetic report definitions; Velocity is detected, never executed.
- **RTF** — `templiqx-rtf` emits bounded interpolated RTF; it does not parse or run Velocity/`$func`.
- **`.xls` input** — explicit non-claim: no BIFF reader. Templiqx emits `.xlsx`/CSV/XML from frozen tabular bindings; aggregation stays in the host/query layer.

Original filenames are preserved (Dutch, as authored in Basenet); the `.xls`
was renamed from `xls OHW per kostenplaats.xls` to `OHW-per-kostenplaats.xls`
to drop the redundant prefix and embedded spaces.
